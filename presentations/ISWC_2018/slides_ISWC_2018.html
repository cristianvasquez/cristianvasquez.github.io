Problem statement

Client/Server architecture, how APIs evolve?

APIs evolution happens in iterations, trying to fulfill (usually) the most common functionality needed by a mass of clients

Problem statement

In these setups, very specific functionality is hard and costly to archieve.
Sometimes not even cost/benefit effective

How to target the long tail of specific
API functionality?

Intuition: Other nodes with similar goals to mine, have already collected relevant functionality, to find following the trails of API evolution

Research Questions

In such a setup

  • How agents expose their API features so they can be collected by others?
  • How can agents discover new functionality without inspecting all nodes?
  • How can API evolution trails be represented so they are suitable for feature discovery?
  • To what degree information derived from provenance trails influence the relevance and precision of API feature discovery?

Hypotheses

  1. An agent can discover and gather features from others relying solely on linked data principles.
  2. It is possible to gather provenance trails to discover services of high relevance without the need of a central service registry.
