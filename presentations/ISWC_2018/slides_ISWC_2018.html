Related work

Semantic Web Services

  • Services described with the help of agreed semantics
  • Usually implemented using registries
  • Allows complex composition of services

Related work

Hypermedia driven APIs

  • Functionality exposed by means of hypermedia controls
  • Clients can discover the interfaces in the Web
  • Allows personalizing content, shaping the APIs according to the data.

Approach

One node (personal agent) can expose
Hypermedia APIs generated dynamically from:

  • Personal and public data
  • Rules and Queries
  • Functionality descriptors and metadata

Approach

Multiple nodes can:

  • Expose and collect functionality from others
  • Evolve their APIs independently

Approach

While APIs evolve, nodes generate
Web API evolution trails used to:

  1. Discover of new functionality
  2. Do mapping between the personal data

Approach

Expectation

  • Nodes can tailor their APIs, adapting them to the data of a user
  • The effort of building APIs is distributed among peers
  • API evolution trails are then useful for bootstrapping API functionality

Research Question

The central question investigated by this thesis is:

How can be supported an API design process in which APIs are adequatelly adapted to the data in control of the user, while interoperability between nodes is not sacrificed?

Hypothesis

Personal agents that exchange API functionality in the Web can build APIs better adapted to the data in control of the users, compared with top down API design approaches used in client-server architectures
